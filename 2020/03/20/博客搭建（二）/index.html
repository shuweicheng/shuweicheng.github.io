Coding Pages 抽风了

前几天刚刚搭建好 Coding Pages / Github Pages + Hexo 的静态网站，没想到还没用到两天，博客突然就打不开了。

本地预览正常，deploy 也正常，而且在 Coding 和 Github 上查看都已经上传成功，即使更换成流量来打开网页依旧是无响应状态。

自己各种百度搜索也找不到答案，心态崩溃，差点想删库重建😑。

最后还是想尝试在网上问问，于是自己上知乎找了下自己之前关注的专栏Hexo+Github个人博客搭建美化 - 知乎专栏 ，试着给博主私信了一下。没想到专栏博主马上回消息，给了一些测试建议😝，结果发现实际上是 Coding Pages 突然崩了。

于是，学到了一点：ping 域名

通过在cmd中ping自己的域名，可以看到实际上反馈的ip地址是多少，这样可以判断到底是Coding page的问题，还是Github Pages的问题？

静态网站的实际连接情况如何，也可以进行检测：

  • Github Pages: ping xxx.github.io 来检测连接网络。
  • Coding Pages: ping 静态网站给的地址来检测连接网络。

当然，也是可以直接ping ip地址的。

Ubuntu云服务器 + Hexo

因为 Coding Pages 时而抽风，Github Pages 访问速度又过慢，所以还是走上了云服务器+Hexo的路，只是这种方法要进行备案。

我选择的是腾讯云服务器，学生价一个月 10 元。建站实际上用 Cent OS 用的更多，但是个人相对熟悉 Ubuntu 一点，所以用 Ubuntu 系统。参照这篇教程HEXO 部署到云服务器详细指南，很快便将博客迁移到了到自己的云服务器上。

这时候不得不谈备案的问题：如果不备案，是无法通过域名访问到过服务器的，但是备案时间比较长，大约二十天。所以这时候你仅能通过 ip 地址访问自己的网站，通过域名访问会提示你需要备案。具体政策服务器接入商会讲解。
